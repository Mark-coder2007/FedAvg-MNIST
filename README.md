# FedAvg on MNIST

## 项目动机

联邦学习要解决的核心问题是,数据没法集中。手机、医院、企业各自攥着自己的数据,出于隐私和监管的原因不能把数据传到一台服务器上统一训练,但大家又都希望有一个又准又通用的模型可以用。这个项目实现的 FedAvg 算法,出自 McMahan 等人二零一七年的工作《Communication-Efficient Learning of Deep Networks from Decentralized Data》,思路很朴素,数据留在本地不动,只把训练好的模型参数传上来,服务器把这些参数加权平均一下再发回去,如此往复。这项工作之所以成为联邦学习的经典起点,是因为它第一次证明了这套朴素的想法在实践中真的能收敛,哪怕客户端之间的数据分布差得很远。

我做这个项目的初衷有两层。第一层是技术上的,我想亲手把这个算法从头到尾实现一遍,而不是只读懂公式。联邦学习最难理解的地方从来不是聚合公式本身,而是数据划分方式对收敛行为的影响,这个直觉只有跑出实验、看着曲线抖动才能真正建立起来。第二层是能力上的,我希望通过这个项目把 Python 和 PyTorch 真正学扎实,并且留下一份自己从零跑通、能讲清楚每一行代码在干什么的作品,而不是一份只停留在概念层面的自我介绍。

## 算法概览

FedAvg 每一轮通信做四件事。服务器从全部客户端里随机挑出一小部分,把当前的全局模型发给它们,每个客户端用自己的本地数据训练若干轮,训练完把参数传回服务器,服务器按各客户端的样本量加权平均这些参数,得到新的全局模型。整个过程反复进行,直到模型在测试集上的表现收敛到满意的水平。

其中最关键的两个超参数是本地训练轮数 E 和本地 batch 大小 B。E 越大,每一轮通信之间客户端本地算得越多,达到同样效果所需的通信轮数就越少,这也是原始工作里强调通信高效的原因。

## 实验设置

模型用的是一个两层卷积加两层全连接的 CNN,输入是二十八乘二十八的灰度图,输出十个类别的得分。

超参数按经典配置对齐,客户端总数100,每轮参与比例百分之十，也就是10个客户端,本地训练5轮,本地 batch 大小10,学习率0.01。

数据划分做了两种,一种是 IID,把训练集完全打乱后均分给一百个客户端,每个客户端拿到的数字分布和整体几乎一样。另一种是病态 Non-IID 划分,先把训练集按标签排序切成两百个小块,再打乱分给一百个客户端,每人两块,这样绝大多数客户端手里只有一到两种数字,这是一个刻意构造的极端场景,用来检验 FedAvg 在数据高度不均衡时还能不能收敛。

## 复现结果

IID 设置下,准确率上升得很快,前十几轮就能冲上百分之九十五以上,曲线整体平滑。切到 Non-IID 之后,收敛明显变慢,曲线在中间阶段抖动幅度也更大,这符合预期,因为每一轮被选中的十个客户端,各自数据分布差异很大,单看某一轮的聚合结果,有可能会因为这一轮选到的客户端恰好都偏向某几个数字而暂时变差,下一轮又被拉回来。这个抖动本身就是联邦学习里数据异质性带来的核心挑战,也是后续一大批改进工作想要解决的问题。

## 和原始结果的对比

原始工作里报告这个 CNN 在 MNIST 上最终能到百分之九十九左右的测试准确率,并且以达到目标准确率所需的通信轮数作为效率指标,IID 下是几十轮量级,病态 Non-IID 下是几百轮量级。我这边跑出来的具体轮数会和它有出入,一部分原因是原始实验对每组配置单独调过学习率等细节,而我这里为了保持对照的公平性,所有设置用了同一组超参数,没有单独调优,这一点我觉得比强行凑数字更值得如实写出来。

## 代码结构

整个实现大约一百六十行,单文件,分成几个部分,模型定义,数据划分,本地训练,服务器端聚合,评估,主流程。每个部分职责单一,方便对照算法流程逐段核对。

## 我在实现过程中的一些想法

关于 Non-IID 下收敛曲线的抖动机制。 IID 与 Non-IID 两条曲线最直观的差异是后者收敛更慢、中段抖动更剧烈。这一抖动的根源可以定位到采样与划分的交互。每轮从一百个客户端中随机抽取十个参与聚合,在 IID 下,无论抽到哪十个,它们合并后的数据分布都接近全局分布,聚合方向稳定;而在病态 Non-IID 下,每个客户端仅覆盖一到两类数字,某一轮抽中的十个客户端可能合计只覆盖了 0 到 4,聚合出的全局模型便向这几类偏移,下一轮抽到另一批客户端又将其拉回。因此抖动本质上来自每轮参与者类别覆盖的随机波动,而非训练本身的不稳定。这一现象也解释了后续工作为何普遍在本地目标中引入对全局模型的约束项,以抑制本地更新的漂移。

关于聚合前的模型拷贝。 实现服务器端聚合时,第一个要想清楚的问题是每个客户端应该拿到一份什么样的模型。最初我写成 model = global_model,准确率始终上不去。原因在于 Python 的赋值只是让 model 和 global_model 指向同一个对象,十个客户端在本地训练时改的是同一份参数,彼此覆盖,最终收集到的十份 state_dict 实际指向同一块内存,加权平均相当于一份参数与自身求平均,根本不存在多客户端独立训练这一前提。改用 copy.deepcopy 后立即恢复正常。这个问题让我意识到,联邦学习数据不出本地这一承诺并非由聚合公式保证,而是由对象语义保证——模型如何下发、参数如何回传、服务器如何聚合,每一步都必须对应到真实分布式场景中数据与参数各自的流向,任何一处退化成引用,隐私边界在代码层面就已经被打破了。
